Apa Itu Tipe Data?

Analoginya: Sebelum menyimpan air, kita butuh botol. Sebelum menyimpan buku, kita butuh rak. Sebelum menyimpan data di database, kita harus tentukan Tipe Datanya!

Definisi: Tipe data adalah pengelompokan data yang memberi tahu MySQL:

  • Jenis nilai apa yang bisa disimpan di sebuah kolom.
  • Berapa besar ruang penyimpanannya.
  • Operasi apa yang bisa dilakukan terhadap data tersebut.

Mengapa Penting? Agar penyimpanan data EFISIEN dan proses pencarian/penghitungan menjadi CEPAT dan AKURAT.

Keluarga Tipe Data Teks (String)

Untuk menyimpan data berupa kumpulan karakter (huruf, angka, simbol).

Tipe Data Penjelasan Analogi Contoh
CHAR(n) Teks dengan panjang pasti. Hemat jika isi selalu penuh. Tempat Telur (ukurannya pasti) CHAR(5) ‚Üí 'AB123' (Kode)
VARCHAR(n) Teks dengan panjang bervariasi. PALING SERING DIPAKAI! Tas Jinjing (sesuai isinya) VARCHAR(100) ‚Üí 'Budi Santoso' (Nama)
TEXT Untuk teks yang sangat panjang. Kontainer (untuk barang banyak) Deskripsi produk, artikel blog.

Pengenalan PRIMARY KEY

Apa Itu Primary Key?
Identifier (pengenal) unik untuk setiap baris data dalam sebuah tabel.

Analoginya:
Setiap siswa di sekolah punya NIS (Nomor Induk Siswa) yang unik dan tidak mungkin sama. NIS inilah Primary Key-nya.

Sifat Wajib Primary Key:

  1. UNIK: Nilainya harus berbeda untuk setiap baris.
  2. TIDAK BOLEH KOSONG (NOT NULL): Harus selalu memiliki nilai.
  3. Hanya ada SATU dalam satu tabel.
Slide 7 of 14

Cara Membuat Primary Key

Praktik Terbaik:
Gunakan kolom id bertipe INT dengan atribut AUTO_INCREMENT.

  • INT: Efisien untuk diproses.
  • AUTO_INCREMENT: MySQL otomatis memberikan angka berurutan (+1) untuk data baru. Kita tidak perlu masukkan manual.

Contoh Syntax:

CREATE TABLE nama_tabel (
    id INT PRIMARY KEY AUTO_INCREMENT, -- Primary Key di sini!
    nama_column Lainnya TipeData_Lain
);
Slide 8 of 14

Contoh Tabel Lengkap dengan Primary Key

Mari buat tabel siswa:

CREATE TABLE siswa (
    id_siswa INT PRIMARY KEY AUTO_INCREMENT, -- PRIMARY KEY
    nis CHAR(10) UNIQUE, -- Unik tapi bukan PK
    nama_depan VARCHAR(50) NOT NULL,
    nama_belakang VARCHAR(50),
    jenis_kelamin ENUM('L', 'P'),
    tanggal_lahir DATE,
    rata_rata_nilai DECIMAL(5,2),
    aktif BOOLEAN DEFAULT TRUE,
    dibuat_pada TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Slide 10 of 14

Mengapa Pakai id INT? Kenapa NIS Tidak Jadi PK?

Alasan menggunakan id INT AUTO_INCREMENT sebagai PK:

  1. Efisiensi: INT (4 byte) lebih cepat diproses & dirujuk daripada CHAR(10) (10 byte).
  2. Stabil: NIS adalah aturan bisnis. Jika format NIS berubah, mengubah Primary Key itu sangat rumit. id integer tidak pernah berubah.
  3. Privacy: id integer tidak punya makna publik, lebih aman.

Kesimpulan: id untuk database, nis untuk dunia nyata.

Slide 11 of 14

Ringkasan & Tips Praktis

  1. Wajib Primary Key: Setiap tabel harus punya Primary Key. Gunakan INT AUTO_INCREMENT.
  2. Pilih Tipe Data yang Tepat:
    • Teks? VARCHAR
    • Angka Bulat? INT
    • Uang/Harga? DECIMAL
    • Tanggal? DATE atau DATETIME
    • Pilihan Tetap? ENUM
  3. Selalu tentukan panjang (n) yang cukup untuk VARCHAR dan CHAR.
