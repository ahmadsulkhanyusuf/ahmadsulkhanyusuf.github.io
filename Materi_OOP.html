Object Oriented Programming (OOP) Dasar

Mengenal paradigma pemrograman berbasis objek dengan PHP untuk struktur kode yang lebih baik.

Pengantar: Analogi Sistem Hero

Menggunakan analogi karakter atau Hero dalam game adalah cara termudah untuk memahami konsep OOP karena setiap elemennya sangat relevan dengan mekanik permainan. Berikut adalah penerapan konsep OOP PHP menggunakan analogi sistem Hero:

1. Class dan Object

Class adalah daftar status dan kemampuan umum seorang Hero. Object adalah hero spesifik yang kita pilih untuk dimainkan.

<?php
class Hero {
    public $nama;
    public $darah;
    public $mana;

    public function __construct($nama, $darah, $mana) {
        $this->nama = $nama;
        $this->darah = $darah;
        $this->mana = $mana;
    }

    public function gunakanSkill() {
        return "$this->nama menggunakan skill dasar!";
    }
}

// Membuat object hero spesifik
$layla = new Hero("Layla", 2500, 400);
$balmond = new Hero("Balmond", 4500, 0);
?>

2. Inheritance (Pewarisan)

Kita bisa membuat kategori hero yang lebih spesifik (Sub-class) yang mewarisi sifat dasar dari class Hero.

<?php
class Mage extends Hero {
    public $magicDamage;

    public function __construct($nama, $darah, $mana, $magic) {
        parent::__construct($nama, $darah, $mana); // Ambil construct dari parent
        $this->magicDamage = $magic;
    }

    // Overriding: Mengubah perilaku method parent khusus untuk Mage
    public function gunakanSkill() {
        return "$this->nama mengeluarkan serangan meteor (Magic Damage: $this->magicDamage)!";
    }
}

$vale = new Mage("Vale", 2200, 600, 500);
echo $vale->gunakanSkill();
?>

3. Encapsulation (Enkapsulasi)

Kita tidak ingin darah hero diubah sembarangan oleh sistem luar tanpa melalui logika tertentu (misalnya, darah tidak boleh lebih dari batas maksimal).

<?php
class Assassin extends Hero {
    private $exp = 0; // Tidak bisa diakses langsung dari luar

    public function tambahExp($jumlah) {
        $this->exp += $jumlah;
        if ($this->exp >= 100) {
            return "$this->nama Level Up!";
        }
    }
}
?>

4. Polymorphism (Polimorfisme)

Semua hero punya cara berbeda untuk melakukan aksi yang sama, misalnya "Attack". Kita bisa menggunakan Interface untuk memastikan setiap hero memiliki method tersebut.

<?php
interface AksiHero {
    public function attack();
}

class Archer implements AksiHero {
    public function attack() {
        return "Menembakkan panah dari jauh!";
    }
}

class Fighter implements AksiHero {
    public function attack() {
        return "Memukul dengan jarak dekat!";
    }
}

// Fungsi yang bisa menerima hero jenis apa saja selama mereka punya interface AksiHero
function mulaiSerangan(AksiHero $hero) {
    echo $hero->attack();
}

mulaiSerangan(new Archer()); // Output: Menembakkan panah...
mulaiSerangan(new Fighter()); // Output: Memukul dengan jarak dekat...
?>

Tabel Analogi OOP Game

Konsep OOP Analogi dalam Game
Class Template Hero (Stats, Skill Slot).
Object Hero yang sedang spawn di dalam map.
Property HP, Mana, Attack Speed, Movement Speed.
Method Skill 1, Skill 2, Ultimate, Recall.
Inheritance Role (Mage, Tank, Marksman) yang semuanya berasal dari class Hero.
Encapsulation Status Buff atau Cooldown yang dikelola internal oleh sistem hero.

Kesimpulan

Dengan struktur ini, kode program menjadi lebih rapi. Jika ingin menambah jenis hero baru (misalnya Healer), Anda cukup membuat class baru yang melakukan extends Hero tanpa harus menulis ulang kode dasar dari awal.
